Interview Application – Overview & Explanation

Purpose: A small, deployable demo built to show the skills needed for the AudienceView Technical Solutions Expert role. It mimics a live-events ticketing system and runs entirely in Docker.

In one sentence: The app is a ticketing-style demo with a responsive web UI (HTML, styles.css, Bootstrap, vanilla JS), a Node.js API, and a SQL Server database—all started with one Docker command—so you can show SQL Server, T-SQL, Node.js, ETL, and Docker in a single project.

1. What the app does

  • Shows a stats strip below the hero: “X events · Y venues · Z tickets” (counts update when data loads).
  • Shows a list of events (e.g. concerts, games) with their venues and dates; each section has a refresh button.
  • Shows Events & ticket sales: each event with tickets sold and revenue (API: JOIN + GROUP BY); “View tickets” expands to show customer, quantity, total per event; close via button or Esc.
  • Shows venues (name, city, capacity) and tickets (event, customer, quantity, total).
  • Lets you add an event and sell a ticket; success shows a toast popup and refreshes the lists.
  • Lets you load more events via ETL: paste a JSON array, click Run ETL; the API validates, transforms, and inserts into SQL Server.

Everything is read from and written to a SQL Server database named TicketingDemo, created and filled automatically on first run. The frontend uses styles.css for layout, cards, tables, stats strip, refresh buttons, success toasts, the ticket–event module, and responsive design (tablet and mobile breakpoints, touch-friendly controls).

2. How it's built (architecture)

Three parts: Frontend (HTML, styles.css, Bootstrap, JavaScript — responsive, single page), API (Node.js, Express, T-SQL; endpoints for events, venues, tickets, /api/events/ticket-summary, ETL), Database (SQL Server, Venues, Events, Tickets). Local: docker-compose up -d → http://localhost:3000. EC2: docker-compose -f docker-compose.prod.yml --env-file .env up -d --build (see DEPLOY-EC2.md).

3. How to run it

Local (Docker):

  1. Open a terminal in the Interview application folder.
  2. Run: docker-compose up -d
  3. Wait ~30–45 seconds the first time.
  4. Open in a browser: http://localhost:3000

To stop: docker-compose down

Production (EC2): Copy or clone the project to the server, create .env from .env.example, then run docker-compose -f docker-compose.prod.yml --env-file .env up -d --build. App is on port 80. Full steps in DEPLOY-EC2.md.
